# Security Policy

This repository is a research and education scaffold, not a production telecom deployment. Security review is still welcome, but the current repository mainly contains documentation, templates and placeholder manifests.

## Reporting

Please do not open public issues for potentially sensitive security findings. Instead, contact the maintainers privately using the reporting channel defined by the project owner.

## Current Scope

Security-relevant review is most useful for:

- unsafe shell scripting;
- accidental credential exposure;
- overly permissive example configuration;
- misleading security claims in documentation.

## Out of Scope

The repository does not currently ship a production-ready platform, so absence of hardened operational controls should be understood as a known limitation rather than an implied feature gap.
